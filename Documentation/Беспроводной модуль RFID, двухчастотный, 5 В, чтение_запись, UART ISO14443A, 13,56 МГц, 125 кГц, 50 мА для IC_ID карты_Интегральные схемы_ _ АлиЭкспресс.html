    1. Описание:

     

    7941 Вт-это двухдиапазонный модуль считывания и записи с помощью нескольких протоколов. Набор IC и ID для чтения и записи. Он может читать различные IC и ID-карты. Поддержка 1K,UID карты, IC карты, T5577 ID карты.

     

    2. Особенности:

     

    1>. Напряжение: 5 В постоянного тока

    2>. Ток: 50 мА

    3>. Расстояние: EM>5 см

    4>. Размер: 47mmX26mmX5mm

    5>. Интерфейс: UART

    6>. Поддержка чипов: ISO/IEC 14443 , NTAG, MF1xxS20, MF1xxS70, MF1xxS50

    7>. EM4100, T5577 функция чтения и записи

    8>. Рабочая температура: -25 ~ 85 по Цельсию

    2>.RX: получить pin

    3>.TX: штифт передачи

    4>.GND: заземление источника питания

    5>.IO: определение

    Отправка:

    1). 0x10 считывание номера UID

    Передает данные:

    1). Чтение указанного раздела: первый байт данных представляет собой секцию; Второй байт означает определенный блок отрасли; Третий байт означает пароль группы A или B (0x 0A/0x0B); затем поставляется с паролем в 6 байт

    2). Заданный участок записи: первый байт данных представляет собой секцию; Второй байт означает определенный блок отрасли; Третий байт означает пароль группы A или B (0x 0A/0x0B); затем он поставляется с паролем в 6 байтов и блоком данных в 16 байтов

    3). Изменить пароль: первый байт означает определенный сектор; Второй байт означает пароль группы A или B (0x 0A/0x0B); Затем Он поставляется со старым паролем в 6 байтов и новым паролем

    Приема данных:

    Чтение указанного формата данных о возврате в сектор, первый байт-это сектор; Второй байт-это определенный блок в секторе; Затем он поставляется с данными о блоке в 16 байт

    7>. Проверка XOR: результат проверки других байтов, за исключением заголовка протокола

    Описание:

     

    7941D-это модуль считывателя RFID. Он интегрирует Высокопроизводительные радиочастотные схемы и антенны. Он может поддерживать считывание ID и IC-карты.

     

    Параметры:

     

    1>. Название товара: 7941D RFID беспроводной модуль для чтения

    2>. Рабочее напряжение: 5 В постоянного тока

    3>. Рабочий ток: 40 мА

    4>. Частота считывания: 125 кГц и 13,56 МГц

    5>. Коммуникационный интерфейс: UART(3 контакта)

    6>. Расстояние считывания: около 8 см (для карты EM4100)

    7>. Расстояние считывания: около 5 см (для S50 S70,Desfire,FM1208 карты)

    8>. Рабочая температура:-20 ℃ ~ 85 ℃

    9>. Рабочая влажность: 0% ~ 95% относительной влажности

    10>. Размер: 47*26*5,3 мм

     

    Интерфейс Описание:

     

    1>.VCC: вход для положительного рабочего напряжения.

    2>.GND: вход для отрицательного рабочего напряжения.

    3>.D0: это TXD UART. Модуль просто может отправить данные.

    Примечание: у модуля только 3 кабеля. Он просто использует TXD UART.

     

    Протокол UART Введение:

     

    1>. Заголовок протокола: 0x02

    2>. Длина данных: 0x09

    3>. Тип карты: 0x01 (больше см. Тип карты)

    4>. Данные карты: SN0 ~ SN3

    5>. Проверка XOR: 1 байт. Результат проверки других байтов, за исключением заголовка протокола

    6>. Конец данных: 0x03

     

    Тип карты (модуль не может читать все карты):

     

    1>.0x02:EM4100 (поддержка)

    2>.0x01:1K (не поддерживает)

    3>.0x03:4K (не поддерживает)

    4>.0x10:HID Карта (не поддерживает)

    5>.0x11:T5567 (не поддерживает)

    6>.0x20:ID-карта (не поддерживается)

    7>.0x21:ISO14443B (не поддерживает)

    8>.0x22:FELICA (не поддерживает)

    9>.0x30:15693 бирка (не поддерживается)

    10>.0x50:CPU карта (не поддерживает)

    11>.0x51: информация по секторам (не поддерживается)

    12>.0xFF: данные клавиатуры (не поддерживаются)

     

    Пример:

     

    Если приемник данных 02 0A 02 2E 00 B6 D7 B5 F2 03.It-это значит, что:

    1>. Первый байт 0x02-это флаг запуска данных.

    2>. Второй байт 0x0A-Общая длина данных. Он включает старт и конец байта.

    3>. Третий байт 0x02 означает, что тип карты EM4100.

    4>. 4-8-й байт (0x2E 0x00 0xB6 0xD7 0xB5)-это номер карты. 4-й байт 0x2E-это скрытый номер карты.

    5>. 9-байтный 0xF2-это XOR контрольный байт.

    6>. 10-й байт-это конечный флаг данных.

     

    Применение:

     

    1>. Посещаемости

    2>. Контроль доступа

    3>. Промышленный контроль

     

    Примечание:

     

    1>. Просто нужно три провода.

    2>. Модуль просто может отправить данные.

    3>. Пользователю необходимо написать код в соответствии с вводом протокола UART, как показано на рисунке.

    4>. Пожалуйста, держите подальше от источников помех.
